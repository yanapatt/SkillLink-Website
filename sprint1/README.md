**Data Structure**
แบ่ง เป็น
ระบบผู้ใช้ = Linked List เพราะ มีการเปลี่ยนแปลงบ่อย  
ระบบกิจกรรมอาสา =  Priority Queue  เพราะสามารถจัดกิจกรรมตามเวลาที่ต้องการได้  
ระบบจับคู่ = graph เพราะ ใช่ในการจับคู่ ระหว่างอาสา กับ ผู้ที่ต้องการความช่วยเหลือ
ระบบสะสมคะแนน = ใช้ Linked List เพราะคะแนนมีเพิ่มมีลดตลอด
(อยู่ระหว่างพิจารณาในการเลือกใช้ Data Structure)
