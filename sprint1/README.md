**Data Structure**
แบ่ง เป็น
ระบบผู้ใช้ = Linked List เพราะ มีการเปลี่ยนแปลงบ่อย  
ระบบกิจกรรมอาสา =  Priority Queue  เพราะสามารถจัดกิจกรรมตามเวลาที่ต้องการได้  
ระบบจับคู่ = graph เพราะ ใช่ในการจับคู่ ระหว่างอาสา กับ ผู้ที่ต้องการความช่วยเหลือ
ระบบสะสมคะแนน = ใช้ Linked List เพราะคะแนนมีเพิ่มมีลดตลอด
(อยู่ระหว่างพิจารณาในการเลือกใช้ Data Structure)

**อธิบายกระบวนการทำงาน โดยใช้ Process, Methods, and Tools อย่างไร**
- Process
1.   **Requirements**
    *   การระดมความคิดของคิดข้อมูลเกี่ยวกับความต้องการของผู้ใช้ 
2.  **พัฒนาต้นแบบ (Prototype Development)**
    *   ออกแบบ UI/UX เพื่อให้เหมาะกับผู้ใช้ทุกวัย และทำให้ระบบเข้าใจง่าย
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   พัฒนาเว็บแอปที่สามารถจับคู่ระหว่างอาสาสมัครและผู้ต้องการความช่วยเหลือ พร้อมระบบสะสมแต้ม รีวิว และแจ้งเตือนกิจกรรม
4.  **ทำเอกสารประกอบโครงงาน**
    *   สร้างเอกสารโครงการเพื่อบันทึกแนวคิด วิธีการพัฒนา และแนวทางการใช้งาน
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   บันทึกสรุปปัญหาและแนวทางแก้ไขของทีมในแต่ละสัปดาห์
- Method
1.  **Requirements**
    *   ใช้ requirements ที่คิดมาได้มาพิจารณากันในกลุ่ม
2.  **พัฒนาต้นแบบ**
    *   ใช้เครื่องมือออกแบบ UI/UX เพื่อสร้างต้นแบบเว็บแอปแบบ Interactive
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   ใช้แพลตฟอร์มที่รองรับการพัฒนาและทดสอบระบบ
4.  **ทำเอกสารประกอบโครงงาน**
    *   ใช้โปรแกรมจัดการเอกสารที่รองรับการทำงานร่วมกันแบบออนไลน์
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   ใช้กล้องวิดีโอคุณภาพสูงและแพลตฟอร์มแชร์วิดีโอ
- Tools
1.  **Requirements**
    *   ใช้ **Microsoft Azure** ในการทำงานร่วมกัน
2.  **พัฒนาต้นแบบ**
    *   ใช้ **Visual studio code**  ในการออกแบบ UI/UX
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   ใช้ **Visual studio code** สำหรับการทำงานร่วมกัน
    *   ใช้ **Microsoft Azure** ในการพัฒนา Backend
4.  **ทำเอกสารประกอบโครงงาน**
    *   ใช้ **Microsoft Azure** ในการจัดทำเล่มโครงงาน
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   ใช้กล้องวิดีโอ
    *   ใช้ **YouTube หรือ Google Drive** ในการแชร์วิดีโอ
