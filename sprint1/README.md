**ที่มาของปัญหาและความสำคัญ**

**จุดประสงค์ของโครงงานและประโยชน์ที่คาดว่าจะได้รับ**

**ขอบเขตของโครงงาน**
1.  สำหรับระบบกิจกรรมจิตอาสาต่าง ๆ ภายในเว๊ปไซต์ เช่น การจับคู่คนที่ต้องการความช่วยเหลือกับอาสาสมัคร และการแจ้งเตือนกิจกรรมจิตอาสาที่กำลังจะเกิดขึ้นในพื้นที่ใกล้เคียง ที่ขึ้นเกิดภายในพื้นที่กรุงเทพมหานคร
2.  มีระบบแสดงความน่าเชื่อถือของบัญชีอาสาสมัคร โดยอ้างอิงจากคะแนนความพึงพอใจและการรีวิวจากผู้ใช้ทั่วไป
3.  มีระบบในการขอสิทธิการเข้าถึงตำแหน่งของผู้ใช้งาน เพื่อเข้าถึงตำแหน่ง ณ ขณะนั้น
4.  สำหรับระบบบัญชีผู้ใช้ทั่วไปและบัญชีของอาสาสมัคร จะมีวิธีการล็อคอินและฟังก์ชันการใช้งานเว๊ปไซต์ที่แตกต่างกัน

**Data Structure**
แบ่งเป็น
- ระบบผู้ใช้ = Linked List เพราะ มีการเปลี่ยนแปลงบ่อย  
- ระบบกิจกรรมอาสา =  Priority Queue  เพราะสามารถจัดกิจกรรมตามเวลาที่ต้องการได้  
- ระบบจับคู่ = graph เพราะ ใช่ในการจับคู่ ระหว่างอาสา กับ ผู้ที่ต้องการความช่วยเหลือ
- ระบบสะสมคะแนน = ใช้ Linked List เพราะคะแนนมีเพิ่มมีลดตลอด
- (อยู่ระหว่างพิจารณาในการเลือกใช้ Data Structure)

**Functional Requirement and Non-Functional Requirement**

- Functional Requirement
1.  ระบบของเว๊บไซต์ต้องสามารถจับคู่ผู้ที่ต้องการความช่วยเหลือกับอาสาสมัครได้
2.  ต้องมีบัญชีแยกอย่างชัดเจน ได้แก่ผู้ใช้ทั่วไปและอาสาสมัคร  
3.  เว๊ปไซต์ต้องมีระบบสะสมแต้มสำหรับอาสาสมัครเพื่อนำไปแลกของรางวัลต่าง ๆ ได้
4.  ผู้ใช้ทั่วไปต้องสามารถรีวิวและให้คะแนนความพึงพอใจสำหรับอาสาสมัครได้
5.  ระบบของเว๊บไซต์ต้องสามารถแจ้งเตือนกิจกรรมจิตอาสาที่กำลังจะเกิดขึ้นในพื้นที่ใกล้เคียงได้
6.  ผู้ใช้ทั่วไปต้องสามารถโพสต์รูปผู้ที่ต้องการความช่วยเหลือเพื่อให้อาสาสมัครมาอ่านรายละเอียดได้

- Non Functional Requirement
1.  ผู้ใช้ต้องการเห็นหน้าตา UI ของเว๊ปไซต์ที่สวยงาม และมีประสบการณ์ใช้งานเว๊บไซต์ที่ดี UX
2.  เว๊บไซต์ต้องมีความใช้งานง่าย เหมาะสมกับทุกเพศทุกวัย
3.  เว๊บไซต์ต้องมีความเสถียร ทนทานต่อจำนวนผู้ใช้ที่เยอะ
4.  หน้าจอเว๊บไซต์ต้องสามารถรองรับได้ทุกระดับความละเอียด ทั้งหน้าจอแนวตั้งและแนวนอน 
5.  ระบบของเว๊บไซต์ต้องมีความปลอดภัย สามารถยืนยันตัวตนได้ว่าใครคืออาสามัครจริง ๆ หรือมิจฉาชีพที่แฝงตัวมา

**อธิบายกระบวนการทำงาน โดยใช้ Process, Methods, and Tools อย่างไร**
- Process
1.   **Requirements**
    *   การระดมความคิดของคิดข้อมูลเกี่ยวกับความต้องการของผู้ใช้ 
2.  **พัฒนาต้นแบบ (Prototype Development)**
    *   ออกแบบ UI/UX เพื่อให้เหมาะกับผู้ใช้ทุกวัย และทำให้ระบบเข้าใจง่าย
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   พัฒนาเว็บแอปที่สามารถจับคู่ระหว่างอาสาสมัครและผู้ต้องการความช่วยเหลือ พร้อมระบบสะสมแต้ม รีวิว และแจ้งเตือนกิจกรรม
4.  **ทำเอกสารประกอบโครงงาน**
    *   สร้างเอกสารโครงการเพื่อบันทึกแนวคิด วิธีการพัฒนา และแนวทางการใช้งาน
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   บันทึกสรุปปัญหาและแนวทางแก้ไขของทีมในแต่ละสัปดาห์
- Method
1.  **Requirements**
    *   ใช้ requirements ที่คิดมาได้มาพิจารณากันในกลุ่ม
2.  **พัฒนาต้นแบบ**
    *   ใช้เครื่องมือออกแบบ UI/UX เพื่อสร้างต้นแบบเว็บแอปแบบ Interactive
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   ใช้แพลตฟอร์มที่รองรับการพัฒนาและทดสอบระบบ
4.  **ทำเอกสารประกอบโครงงาน**
    *   ใช้โปรแกรมจัดการเอกสารที่รองรับการทำงานร่วมกันแบบออนไลน์
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   ใช้กล้องวิดีโอคุณภาพสูงและแพลตฟอร์มแชร์วิดีโอ
- Tools
1.  **Requirements**
    *   ใช้ **Microsoft Azure** ในการทำงานร่วมกัน
2.  **พัฒนาต้นแบบ**
    *   ใช้ **Visual studio code**  ในการออกแบบ UI/UX
3.  **เขียน Code และทดสอบแอปพลิเคชัน**
    *   ใช้ **Visual studio code** สำหรับการทำงานร่วมกัน
    *   ใช้ **Microsoft Azure** ในการพัฒนา Backend
4.  **ทำเอกสารประกอบโครงงาน**
    *   ใช้ **Microsoft Azure** ในการจัดทำเล่มโครงงาน
5.  **บันทึกวิดีโอการประชุม Retrospective**
    *   ใช้กล้องวิดีโอ
    *   ใช้ **YouTube หรือ Google Drive** ในการแชร์วิดีโอ
